Get started

Jenkins X provides automated CI+CD for Kubernetes with Preview Environments on Pull Requests using Tekton as the underlying pipeline engine.

 

Step 1: Provision your Kubernetes Cluster

Provisioning your Kubernetes cluster is easy. Jenkins X does not really care how you provision your cluster, however there are many resources that are provisioned, so we recommend using the Terraform Modules we've made available

  • GCP
  • AWS (coming soon)
  • Azure (coming soon)

Use GKE Terraform Module

1 . Create a main.tf file and add the code snippet below.

{{}} module "jx" { source = "jenkins-x/jx/google" version = "1.2.1" # override default parameter values here } {{}}

This setup uses defaults parameter values. You can modify them, more details here.

2 . Provision the cluster using Terraform. The output of this command will include a file jx-requirements.yml which
you will use in Step 3: Install Jenkins X on your cluster

{{}} $ terraform init && terraform apply {{}}

Use the AWS Terraform Module

1 . Create a main.tf file and add the code snippet below.

{{}} module "eks-jx" { source = "jenkins-x/eks-jx/aws" vault_user="" } {{}} This setup uses defaults parameter values. You can modify them, more details here.
IAM User

Specify existing IAM User

You will need to provide a existing IAM user name for Vault. The specified user's access keys are used to authenticate the Vault pod against AWS. The IAM user does not need any permissions attached to it. This Terraform Module creates a new IAM Policy and attaches it to the specified user.
For more information refer to Configuring Vault for EKS.

2 . Provision the cluster using Terraform. The output of this command will include a file jx-requirements.yml which
you will use in Step 3: Install Jenkins X on your cluster

{{}} $ terraform init && terraform apply {{}}

Step 2: Install the Jenkins X CLI

The first step in using Jenkins X is installing the Jenkins X binary jx for your operating system.

  • OS X
  • Linux
  • Windows

Using Homebrew

{{}} brew install jenkins-x/jx/jx {{}}

Using Curl

{{}} # Download the `.tar` file, unarchive it. curl -L "https://github.com/jenkins-x/jx/releases/download/$(curl --silent https://api.github.com/repos/jenkins-x/jx/releases/latest | jq -r '.tag_name')/jx-darwin-amd64.tar.gz" | tar xzv "jx" # Add `jx` to your path sudo mv jx /usr/local/bin # verify installation jx --version {{}}
  {{}} # Download the `.tar` file, and unarchive it in a directory where you can run the `jx` command. curl -L "https://github.com/jenkins-x/jx/releases/download/$(curl --silent https://api.github.com/repos/jenkins-x/jx/releases/latest | jq -r '.tag_name')/jx-linux-amd64.tar.gz" | tar xzv "jx" # if you don't have `jq` installed: curl -L "https://github.com/jenkins-x/jx/releases/download/$(curl --silent "https://github.com/jenkins-x/jx/releases/latest" | sed 's#.*tag/\(.*\)\".*#\1#')/jx-linux-amd64.tar.gz" | tar xzv "jx" # Move binary to your path sudo mv jx /usr/local/bin # verify installation jx --version {{}}
 

Install Chocolatey using an Admin

1. Right-click menu:Start\[Command Prompt (Admin)\].

2. At the shell prompt, execute a `powershell.exe` command to download and install the `choco` binary and set the installation path so that the binary can be executed: {{}} @"%SystemRoot%\System32\WindowsPowerShell\v1.0\powershell.exe" -NoProfile -InputFormat None -ExecutionPolicy Bypass -Command "iex ((New-Object System.Net.WebClient).DownloadString('https://chocolatey.org/install.ps1'))" && SET "PATH=%PATH%;%ALLUSERSPROFILE%\chocolatey\bin" {{}} 3. Install Jenkins X using Chocolatey: {{}} choco install jenkins-x {{}} Upgrading Jenkins X via Chocolatey {{}} choco upgrade jenkins-x {{}}

 

 

 

Step 3: Install Jenkins X on your cluster

You have provisioned your Kubernetes cluster and the CLI. You are ready to install Jenkins X on top of the cluster. To do that, you will use the JX Boot CLI command.

What is JX Boot? With JX Boot you Install, configure or upgrade Jenkins X via GitOps and a Jenkins X Pipeline. This pipeline resides within your config git repository, also known as a permanent environment
{{}} # the parameter here, is the requirements yaml file # which was saved in the same directory you executed terraform apply. $ jx boot -r jx-requirements.yml {{}}
